System Architecture

Agents & What They Do

Agent 1 — agent.ts (CPM Reporting Agent)

Runs inside Guild.ai. Primary job: synthesize CPM benchmark data into actionable insights. Secondary job: track its own HITL interactions and ROI over time.

ModeTriggerWhat it does
A — Pipeline Run{ run: true }Fetch latest CPM data from Neon via Malloy, generate report, log run metrics
B — Change Request{ changeRequest: {...} }Record user request to modify report format/content. Logs HITL Type 2 event.
C — Mark Implemented{ markImplemented: {...} }Close a change request loop. Tracks implementation rate.
D — Custom Query{ query: "..." }Run ad-hoc Malloy query against cpm_benchmarks
E — Dashboard{ dashboard: true }Show ROI summary: automation rate, cost per output, human time recovered
F — Reset{ reset: true }Clear state (with confirmation). Preserves Neon data.

Agent 2 — universal-tracker.ts (AI ROI Tracker)

Tracks ROI of ALL AI interactions across every provider (Claude, ChatGPT, Cursor, Gemini, Copilot, etc.). Neon is the source of truth; Guild provides the conversational interface.

ModeTriggerWhat it does
INIT{ init: true }Seeds 7 backfill tasks from this conversation into Neon. One-time, guarded against repeat.
LOG{ log: {...} }Record a new AI interaction. Auto-calculates value from hourly rate if omitted.
CORRECTION{ correction: {...} }Mark that an interaction needed rework. Downgrades first_pass quality score.
DASHBOARD{ dashboard: true }Live ROI summary from Neon: total value, cost, ROI multiple, first-pass %, by provider.
QUERY{ query: "roi_by_provider" }Named Malloy view against ai_interactions (when Malloyyo sync is complete).
RESET{ reset: true, confirm: true }Clear Guild session state. Neon data preserved.

Honest Assessment — What Went Wrong & Better Design

July 4 2026 — Fully resolved. All systems on live Neon.

The interactions source in Malloyyo now reads directly from Neon Postgres via neon.table('public.ai_interactions'). The original blocker was using discrete env vars (NEON_HOST, NEON_DATABASE, etc.) — these weren't set on the Malloyyo deployment, so node-postgres defaulted to localhost:5432 → ECONNREFUSED. The fix, confirmed by Lloyd Tabb (Malloyyo founder): switch to connectionString: { "env": "DATABASE_URL" } in malloy-config.json. DATABASE_URL was already set. MCP query confirmed live July 4 — returned Neon rows including July 4 timestamps.

/api/interactions (Vercel → Neon direct) remains available as a second path — used by universal-tracker's QUERY and DASHBOARD modes, and as a fallback if Guild's sandbox blocks outbound HTTP to Malloyyo.

Why we ended up here

The original architecture was sound but we hit two unforeseen obstacles: (1) Malloyyo turned out to be a multi-tenant SaaS with its own storage model, not a simple file server. (2) Its admin interface requires Google OAuth that isn't currently configured. Neither was knowable without attempting it. The correct response is to route around these blockers rather than keep fighting them.

How to Log Any AI Interaction

# From terminal (any project)
log-ai \
  --provider anthropic \    # anthropic | openai | google | cursor | github | ...
  --tool     cowork \       # cowork | claude-code | chat | cursor | copilot | ...
  --type     code \         # code | document | analysis | testing | research | design
  --hours    2.5 \          # hours this would have taken you manually
  --value    375 \          # optional — auto-calculated from hourly rate if omitted
  --desc    "Rewrote auth module" \
  --project  cpm-agent \    # namespace (default: "default")
  --cost     0.97           # actual API cost in USD (omit for subscription tools)

# Subscription tools (Cursor, Copilot — no per-token cost)
log-ai --provider cursor --tool cursor --type code \
       --hours 1 --desc "Refactored DB layer" \
       --cost-model subscription
